  1. Que es un sistema de control de versiones?

    [Es un sistema o software que permite tener un control sobre los cambios que se van realizando sobre uno o mas archivos a lo largo de la creación de un proyecto. De modo tal que si alguna version posterior tiene dificultades, por ejemplo, puedas tener la posibilidad de volver a una version anterior que era funcional.]

  2. Cual es la diferencia entre git y github?

    [Git es el sistema de control de versiones, que es un software permite tener un control de cambios sobre el código fuente de una aplicación mientras que GitHub GitHub es una página que ofrece un grupo de servicios que facilitan el uso de Git, como por ejemplo hosting de proyectos, facilidades de colaboración, reviews de código, perfiles personales, pull requests, issues entre lo más destacado.]

  3. Cómo creo una rama en git?

    [La forma de crear una nueva rama es con el comando git branch. Ejemplo: git branch nombre_de_la_rama. Para movernos a esa rama se usa el comando git checkout y el nombre de la rama. Ejemplo: git checkout nombre_de_la_rama. Una atajo para poder crear la rama y movernos luego de crearla es: git checkout -b nombre_de_la_rama.]

  4. Cómo clono un repositorio?

    [Dentro de la URL del repositorio de GitHub seleccione el boton verde que dice code, a continuacion aparecerán dos opciones, clone y download zip, seleccionar clone HTTPS y copiar. Desde la terminal y situados en la carpeta donde vamos a clonar el repositorio, ejecutamos el comando git clone + dirección del repositorio que copiamos recientemente.

  5. Cómo puedo bajar los ultimos cambios de un repositorio?

    [Para mantener actualizado el repositorio, el comando que se utiliza para traer los ultimos cambios es git pull + nombre_de_la_rama. Ejemplo: git pull origin master o git pull origin nombre_de_la_rama.

  6. Cómo puedo generar un tag?

    [El comando para generar un tag es: git tag + nombre_del_tag. Ejemplo: git tag v1.0.0. Adicionalemente se puede dejar un comentario de igual manera que se realiza un commit. Ejemplo: git tag -a v1.0.0 -m "Esta es la version 1.0.0"]

  7. Cómo hago una mezcla de ramas?

    [Para mezclar dos ramas hay dos formas: git merge y git rebase. El más utilizado es git merge debido a su estabilidad en materia de reducción de conflictos. Utilizar rebase nos puede ayudar a evitar conflictos siempre que se aplique sobre commits que están en local y no han sido subidos a ningún repositorio remoto. Si no se tiene precaución otra persona podría utilizar cambios dañados. Ejemplo de mezcla de ramas: git checkout master y git merge nombre_de_la_rama.]
